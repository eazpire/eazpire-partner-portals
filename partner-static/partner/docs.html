Partner API Documentation

Machine access for manufacturers on partner.eazpire.com — manage catalog products and fulfill orders.

Authentication Scopes Endpoints Errors Examples Portal API keys

Authentication

Create an eazpire Partner API key in the Partner Portal under API. The full key is shown once.

Send the key on every request:

  • Authorization: Bearer eazpire_mfg_…
  • or X-Eazpire-Partner-Key: eazpire_mfg_…

Portal browser sessions use a cookie instead and have full access (*). Keys are scoped to one manufacturer and never store the raw secret after creation.

Scopes

When creating a key you can pick scopes, or leave defaults (all listed scopes). Use * for full API access.

ScopeAllows
overview:readDashboard / overview stats
company:readRead company profile
company:writeUpdate company profile
products:readList / get products
products:writeCreate / update products and submit for review
orders:readList / get orders and print-file URLs
orders:writeAccept, reject, status, and tracking updates
*All scopes

Endpoints

Base URL: https://partner.eazpire.com. Prefer versioned paths; ?op= aliases still work.

Overview & company

Method / pathScopeNotes
GET /api/v1/overview overview:read Dashboard counts and status
GET /api/v1/company company:read Manufacturer profile + certification progress
POST /api/v1/company company:write Update company fields (same body as portal company form)

Products

Method / pathScopeNotes
GET /api/v1/products products:read Optional ?status=draft|in_review|approved|…
POST /api/v1/products products:write Create draft. Body: { title } required; optional description, category, currency, base_cost_cents, …
GET /api/v1/products/{id} products:read Product + variants + print areas
POST /api/v1/products/{id} products:write Update product metadata
POST /api/v1/products/{id}/submit-review products:write Submit for Eazpire review (requires valid print areas)

Orders

Method / pathScopeNotes
GET /api/v1/orders orders:read Optional ?status=received|accepted|in_production|shipped|…
GET /api/v1/orders/{id} orders:read Order + line items (print files, placement, artifact)
POST /api/v1/orders/{id}/accept orders:write Mark order accepted
POST /api/v1/orders/{id}/reject orders:write Mark order rejected
POST /api/v1/orders/{id}/status orders:write Body: { status } (default in_production)
POST /api/v1/orders/{id}/tracking orders:write Body: { tracking_number, tracking_url? } — sets status shipped
GET /api/v1/orders/{id}/print-file orders:read Optional ?item_id= — signed print file URL

Examples

List products

curl -sS "https://partner.eazpire.com/api/v1/products" \
  -H "Authorization: Bearer eazpire_mfg_YOUR_KEY"

Create a product

curl -sS -X POST "https://partner.eazpire.com/api/v1/products" \
  -H "Authorization: Bearer eazpire_mfg_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"title":"Classic Hoodie","category":"apparel","currency":"EUR","base_cost_cents":1200}'

List open orders

curl -sS "https://partner.eazpire.com/api/v1/orders?status=received" \
  -H "Authorization: Bearer eazpire_mfg_YOUR_KEY"

Accept an order

curl -sS -X POST "https://partner.eazpire.com/api/v1/orders/mord_…/accept" \
  -H "Authorization: Bearer eazpire_mfg_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{}'

Ship with tracking

curl -sS -X POST "https://partner.eazpire.com/api/v1/orders/mord_…/tracking" \
  -H "Authorization: Bearer eazpire_mfg_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"tracking_number":"DE123","tracking_url":"https://track.example/DE123"}'
